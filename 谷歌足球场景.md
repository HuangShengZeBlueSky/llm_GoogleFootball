**运行流程**

1. 观测（Observation）
   - 环境通过 create_environment 提供多种表示，representation 支持 raw（向量信息）或 pixels（图像像素）。([di-engine-docs.readthedocs.io](https://di-engine-docs.readthedocs.io/en/latest/13_envs/gfootball.html))
   - raw 里包含球的位置与速度、队员位置与速度、比赛状态等结构化信息。([di-engine-docs.readthedocs.io](https://di-engine-docs.readthedocs.io/en/latest/13_envs/gfootball.html))
2. 思考（Reasoning）
   - 你的策略网络/算法在每一步根据观测做决策。GRF 支持控制单个或全部球员，并强调高层动作而非低层控制。([grf-marl.readthedocs.io](https://grf-marl.readthedocs.io/en/latest/handbooks/scenarios.html))
3. 行动（Action）
   - 动作空间是 **19 维离散动作**，包含移动、传球、射门、加速、抢断、带球等。([di-engine-docs.readthedocs.io](https://di-engine-docs.readthedocs.io/en/latest/13_envs/gfootball.html))
4. 交互（Interaction loop）

- 标准 Gym 式流程：reset → observations → step(action) 返回下一观测与回报。([di-engine-docs.readthedocs.io](https://di-engine-docs.readthedocs.io/en/latest/13_envs/gfootball.html))

```python
"""3v1 corner-kick training scenario for Google Research Football."""

from . import *  # pylint: disable=wildcard-import


def build_scenario(builder):
    # Episode configuration mirrors the official academy corner setup.
    builder.config().game_duration = 400
    builder.config().deterministic = False
    builder.config().offsides = False
    builder.config().end_episode_on_score = True
    builder.config().end_episode_on_out_of_play = True
    builder.config().end_episode_on_possession_change = False

    # Corner location: right side, near top corner (same as academy_corner).
    builder.SetBallPosition(0.99, 0.41)

    # Left team (attackers).
    builder.SetTeam(Team.e_Left)
    builder.AddPlayer(-1.0, 0.0, e_PlayerRole_GK, controllable=False)
    builder.AddPlayer(1.0, 0.42, e_PlayerRole_LB)  # corner taker
    builder.AddPlayer(0.72, 0.12, e_PlayerRole_CM)
    builder.AddPlayer(0.70, -0.06, e_PlayerRole_CF)

    # Right team (1 defender + keeper).
    builder.SetTeam(Team.e_Right)
    builder.AddPlayer(-1.0, 0.0, e_PlayerRole_GK)
    builder.AddPlayer(-0.75, 0.0, e_PlayerRole_CB)

```

它参考了 academy_corner 与 academy_3_vs_1_with_keeper 的配置风格（时长、是否越位、角球球位、角球手站位等），并将人数缩为 3v1（含守门员）。

**场景要点**

- 球位置：[(0.99, 0.41)](https://file+.vscode-resource.vscode-cdn.net/c%3A/Users/黄胜泽/.vscode/extensions/openai.chatgpt-0.4.69-win32-x64/webview/#)，右侧角球点
- 左队：GK + 3 名进攻（角球手 + 2 个禁区点位）
- 右队：GK + 1 名防守
- 默认不在丢失球权时立即结束回合，便于角球后续演练
